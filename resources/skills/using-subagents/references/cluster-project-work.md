# 集群处理项目

当项目很大、文件多、可按模块并行处理或需要多个专业视角时，读取本文件。

## 分片方式

按结构拆分：前端、后端、工具层、测试、文档、配置；或按模块、目录、文件类型拆分。

按阶段拆分：发现相关代码和模式；比较方案和抽象边界；按文件集合实现；运行局部测试；交叉审查；由主 agent 合并并回归验证。

按注意力焦点拆分：安全、状态管理、用户界面、性能、测试覆盖、维护性。

## 文件边界

- 主 agent 在启动 subagents 前列出每个任务的目标、文件集合、输出格式和测试要求。
- 一个文件同一轮只分配给一个负责修改的 subagent。
- 如果 subagent 发现必须修改未分配文件，应输出原因和建议，由主 agent 重新分片或合并处理。
- 共享接口、schema、lockfile、CI/CD、迁移等容易造成全局影响的文件，优先由主 agent 串行处理，或单独分片后广播新上下文。

## 实现与合并

Subagent 可以并行产出代码、patch、测试或审查意见。主 agent 负责收集变更、检查冲突、按依赖顺序合并、运行相关测试，并在冲突时启动第二轮 targeted prompt。

每个实现 subagent 输出：负责范围、变更文件、diff 摘要、测试结果、风险点、未验证点。

## 汇总格式

```md
## 集群处理结果

- 分片方式：[按模块/角色/文件集合/阶段]
- 已用 subagents：[名称 -> 任务 -> 文件范围]
- 变更清单：[文件 -> subagent -> 状态]
- 交叉审查结果：[审查对 -> Approve/Block]
- 合并冲突：[冲突文件 -> 解决方式]
- 测试结果：[各阶段测试通过/失败]
- 主 agent 决策：[采纳/拒绝/需要二轮]
```
