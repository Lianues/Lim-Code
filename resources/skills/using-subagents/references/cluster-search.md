# 集群搜索

当用户要求多方向搜索、社交媒体挖掘、真实用户体验验证或证据交叉验证时，读取本文件。

## 切分维度

按证据来源拆分：官方文档、GitHub issues/PR/仓库、Hacker News/Reddit/论坛/开发者评论、学术论文/基准/安全报告、中文与英文资料。

按问题立场拆分：支持证据、反例和失败案例、用户吐槽、替代方案、长期维护风险。

按技术面拆分：API 设计、性能、安全、生态兼容性、迁移成本、测试策略。

## 搜索上下文

搜索 subagent 没有对话记忆。prompt 必须保留判断所需上下文：问题域、技术栈、版本、规模、约束、排除范围、证据标准和反例要求。

不要因为“简短”而丢掉关键词、候选方案、失败模式和用户真实目标。不要夹带密钥、私有凭据或未授权私有数据。

## Prompt 要求

每个搜索 subagent 都应收到：明确搜索角度、来源范围、排除范围、证据类型、输出格式、置信度要求和未验证点。要求总结结论，不要逐字复述网页。

## 汇总规则

主 agent 汇总时按证据强度排序：官方规范和代码实现 > 可复现 issues/PR/测试结果 > 多个独立用户的一致反馈 > 单篇博客或单条评论。

不要把搜索结果当作事实本身。若多个来源冲突，启动第二轮 targeted search 或让审查 Agent 专门验证冲突点。
