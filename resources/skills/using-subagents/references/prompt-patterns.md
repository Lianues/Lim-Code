# Subagent Prompt 模式

当需要写给 subagent 的自包含 prompt 时，读取本文件。

## 基本模板

```md
你是一个无状态 subagent。不要假设自己知道本 prompt 之外的对话历史、主 agent 隐含计划、其他 subagent 输出或上一轮结论。

任务：[明确任务]
背景：[必要上下文，或要求读取的文件路径；若涉及上一轮/第一轮，必须列出上一轮结论和阻断项，或指向共享信息库中的对应报告]
输入范围：[路径、搜索来源或资料]
输出产物：[结论 / patch / 测试结果 / 审查意见]
文件边界：[可读路径；若要修改，列出可修改文件集合]
冲突规则：[不要修改未分配文件；必要时提出请求]
证据要求：[文件路径、URL、命令输出、失败复现]
完成标准：[Approve/Block、最多 N 个问题、必要修正]
```

## 对抗审查模板

```md
请从 [角色/角度] 审查以下设计。主动寻找阻断问题，不要迎合候选方案。

候选设计：[内容或文件路径]
必须验证：[列出关键主张]
输出只允许：Approve 或 Block。若 Block，列出最多 5 个阻断点，每个点包含证据和建议修正。
```

## 定向复核模板

```md
你是无状态复核 subagent。不要假设自己知道上一轮对话。

被复核对象：[文件路径或设计摘要]
上一轮结论：[Approve/Block/其他]
上一轮阻断项：[逐条列出，含证据]
本轮修订摘要：[逐条对应说明]
复核范围：只判断本轮修订是否解决上述阻断项，不扩展新审查范围。
缺失检查：如果上一轮结论、阻断项、修订摘要或被复核对象缺失，先输出 REJECT: 缺少复核上下文。
输出：APPROVE 或 REJECT；若 REJECT，只列仍未解决的阻断项。
```

## 集群搜索模板

```md
你是一个无状态搜索 subagent。不要假设自己知道本 prompt 之外的上下文。

搜索任务：[具体问题，含技术栈名、场景约束、规模参数]
搜索来源：[官方文档 / GitHub issues / 论坛 / 学术论文 / 中英文分别]
必须包含：[反例、失败案例、用户吐槽、替代方案]
排除范围：[不要某类来源、不要某时间之前、不要某技术栈]
输出格式：结论、URL、证据类型、置信度、未验证点。不要逐字复述网页。
```

## 项目分片模板

```md
请处理项目中的 [目录/模块/风险范围]。

处理方式：[只读分析 / 产出 patch / 修改指定文件 / 运行测试]
读取范围：[文件路径]
文件边界：[可修改文件集合；若只读则写“无”]
目标：[要回答或实现的问题]
冲突规则：不要修改未分配文件；必要时提出请求。
输出：已读文件、变更文件、diff 摘要、测试结果、风险、未验证点。
```

## 反模式

- “你看一下这个项目”但不给范围。
- “继续上一个 Agent 的工作”但不提供上一个 Agent 的输出。
- “只审查第一轮阻断问题”但不列出第一轮结论、第一轮阻断项，也不指向共享信息库中的上一轮报告。
- subagent 返回了新事实、决策或阻断项，主 agent 没有回写共享信息库。
- 让 subagent 同时搜索、设计、修改、测试、发布，却不给任务边界、文件边界和输出标准。
- 要求 subagent 输出所有原文资料，而不是摘要和证据链接。
